# AI PPT Pipeline（AI PPT 生成流水线）

一种将 AI 驱动的 PPT 生成过程分解为固定顺序的多阶段流水线架构，每个阶段由特定 AI 角色负责，阶段之间通过结构化产物（Markdown → 设计规格 → SVG → PPTX）传递状态。

## 核心设计原则

1. **角色单一职责**：每个流水线阶段由一个专门的 AI 角色承担，拆分"理解→设计→执行→导出"为独立步骤。
2. **单一阻断点 (Single Checkpoint)**：整个流水线只在策略师阶段设置人机确认点，避免在中后期反复打断，最大化自动化效率。
3. **不可合并的后处理**：后处理三步（分页 → SVG 定稿 → 转 PPTX）必须依次执行，因为每步的输出是下一步的输入，合并执行会导致中间态丢失。
4. **执行锁机制 (spec_lock.md)**：策略师确认后将设计决策锁定为机器可读文件，后续阶段每页生成前必须重读，防止设计漂移。

---

## 流水线阶段

```
[源文档] → 格式转换(MD) → 项目初始化 → [模板加载] → 策略师设计 ⛔ → [图片获取] → 执行器生成SVG → 后处理三步骤 → [PPTX输出]
```

- **⛔** 标记唯一的人机确认点

| 阶段 | 产物 | 说明 |
|------|------|------|
| 格式转换 | Markdown | 将 PDF/DOCX/URL 等转为统一文本 |
| 项目初始化 | 目录结构 | 创建标准化的项目文件夹 |
| 模板加载 | 模板文件 | 如有指定则加载企业/学术模板 |
| 策略师设计 | `design_spec.md` + `spec_lock.md` | 8 项设计建议，等待用户确认 |
| 图片获取 | `images/` | AI 生成或网络搜图 |
| 执行器生成 | `svg_output/` + `notes/` | 逐页 SVG 和演讲备注 |
| 后处理 | `svg_final/` → `.pptx` | 分页、定稿、转 PPTX |

---

## 关键约束

- **串行刚性**：步骤不可跳跃、不可并行
- **阶段隔离**：禁止跨阶段打包操作（如在设计阶段就开始写 SVG）
- **角色绑定**：执行器（SVG 生成）必须在主代理中完成，不可委托子代理
- **状态锁**：每页生成前必须重读 `spec_lock.md`，确保颜色/字体/图标一致性

> [!warning] 串行刚性的代价
> 无法根据后续阶段的发现回溯调整设计。如果发现某页内容太多需要增加页数，必须重启整个流水线或接受当前输出。

---

## 适用场景

- 从源文档自动生成结构化演示文稿
- 需要批量生成风格统一的 PPT 的场景
- 非设计人员快速产出专业级演示文稿

---

## 局限与权衡

- **灵活性牺牲**：串行刚性意味着无法根据后续阶段的发现回溯调整设计
- **SVG 中间层开销**：多了一层转换，增加了处理时间
- **Claude Code 强依赖**：目前仅能在 Claude Code 环境中运行
- **模板路径依赖**：用户必须给出完整模板路径才能使用模板，增加了使用门槛
