# VersionControl（版本控制）

版本控制（Version Control）是一种记录文件内容变更的系统，允许开发者回溯历史版本、并行开发和多人协作，而不会互相覆盖对方的修改。

## 定义

版本控制是一种记录文件内容变更的系统，允许开发者回溯历史版本、并行开发和多人协作，而不会互相覆盖对方的修改。

---

## 核心概念

| 概念 | 说明 |
|------|------|
| **仓库（Repository）** | 存储代码和完整变更历史的容器 |
| **暂存区（Staging Area）** | 提交前的临时区域，用于选择要纳入下次提交的变更 |
| **提交（Commit）** | 将暂存区的变更正式写入版本历史，附带说明信息 |
| **远程仓库（Remote）** | 托管在服务器上的仓库副本，用于团队同步 |
| **`.gitignore`** | 指定不需要纳入版本管理的文件和目录（如依赖包、环境变量、日志文件） |

---

## 分布式 vs 集中式

- **分布式（如 Git）**：每个开发者本地拥有完整的历史副本，无需依赖中央服务器，可离线工作
- **集中式（如 SVN）**：所有历史只保存在中央服务器，开发者本地只有当前工作副本

> [!info] 为什么 Git 是分布式的
> Git 由 Linus Torvalds 为 Linux 内核开发而设计，目的是让数千名开发者能在没有中央服务器的情况下并行协作。每个本地仓库都是完整的备份，服务器挂了也不丢历史。
